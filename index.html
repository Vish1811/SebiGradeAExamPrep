Complete Syllabus Breakdown

Phase I has 9 topics. Phase II goes deeper into algorithms, data structures, string manipulation, and OOP. Both phases covered below.

🗄️Database Concepts10%
  • ER Model — entities, attributes, relationships, cardinality
  • Relational Model — relational algebra, tuple calculus
  • Integrity Constraints — primary key, foreign key, domain, referential
  • Normal Forms — 1NF, 2NF, 3NF, BCNF with examples
  • File Organization — sequential, indexed, hashed
  • Indexing — B-trees, B+ trees, dense vs sparse index
  • Transactions — ACID properties, serializability
  • Concurrency Control — locking protocols, deadlock
🔍SQL Queries10%
  • DDL — CREATE, ALTER, DROP, TRUNCATE
  • DML — SELECT, INSERT, UPDATE, DELETE
  • Joins — INNER, LEFT, RIGHT, FULL OUTER, SELF, CROSS
  • Aggregate Functions — COUNT, SUM, AVG, MAX, MIN with GROUP BY / HAVING
  • Set Operations — UNION, INTERSECT, EXCEPT
  • Subqueries — IN, EXISTS, correlated subqueries
  • Views — creation, updatable views, materialized views
  • Nested Queries — multi-level SELECT with conditions
💻Programming — Java / C / C++30%
  • Program Control — iteration, recursion, function calls
  • Scope & Binding — variable scope, static vs dynamic binding
  • Parameter Passing — pass by value vs reference, pointers in C
  • OOP Concepts — encapsulation, abstraction, polymorphism, inheritance
  • Java Specifics — JVM, class hierarchy, interfaces, abstract classes
  • Constructors — default, parameterized, copy constructor, constructor chaining
  • Exception Handling — try/catch/finally, custom exceptions, checked vs unchecked
  • Functional Programming — lambda basics, pure functions concept
🐍Data Analytics — Python / R10%
  • Python Basics — lists, tuples, sets, dictionaries, comprehensions
  • Regex — pattern matching, re module, search/match/findall
  • Pandas — DataFrames, slicing, groupby, merge, pivot, reshape
  • NumPy — arrays, broadcasting, vectorization
  • File Management — reading/writing CSV, JSON, Excel using Python
  • Classes & Functions — OOP in Python, decorators, generators
  • Data Mining — basic concepts of classification, clustering
  • Visualization — charts/graphs using matplotlib/seaborn concepts
⚙️Algorithms for Problem Solving10%
  • Tree Traversals — inorder, preorder, postorder, BFS, DFS
  • Graph Algorithms — BFS, DFS, connected components
  • Spanning Trees — Kruskal's, Prim's MST algorithms
  • Shortest Path — Dijkstra's, Bellman-Ford, Floyd-Warshall
  • Hashing — hash functions, collision handling (chaining, open addressing)
  • Sorting — QuickSort, MergeSort, HeapSort, time complexities
  • Searching — binary search, interpolation search
  • Design Techniques — Greedy, Dynamic Programming, Divide and Conquer
🌐Networking Concepts10%
  • OSI Model — all 7 layers, functions, protocols at each layer
  • TCP/IP Stack — TCP vs UDP, IP addressing, subnetting
  • LAN Technologies — Ethernet (CSMA/CD), Token Ring
  • Application Protocols — DNS, SMTP, POP3, IMAP, FTP, HTTP/HTTPS
  • Network Devices — Hub, Switch, Router, Bridge, Gateway
  • Firewalls — packet filtering, stateful inspection, proxy
  • IP Classes — Class A/B/C, CIDR, NAT, private/public IPs
  • Routing Protocols — RIP, OSPF, BGP concepts
🔐Information & Cyber Security10%
  • CIA Triad — Confidentiality, Integrity, Availability in depth
  • Cyber Attacks — phishing, DDoS, ransomware, MITM, SQL injection, XSS
  • Authentication — MFA, biometrics, tokens, OAuth, Kerberos
  • Cryptography — symmetric (AES, DES), asymmetric (RSA), hashing (SHA)
  • PKI — digital certificates, CA, SSL/TLS handshake
  • Network Security — IDS/IPS, VPN, DMZ, honeypots
  • Software Development Security — SDLC security, OWASP Top 10
  • Audits — network audits, systems audits, penetration testing basics
🏭Data Warehousing5%
  • ETL Process — Extraction, Transformation, Loading steps
  • Data Cleaning — handling nulls, duplicates, outliers
  • Data Cube — OLAP operations: slice, dice, drill-down, roll-up
  • Data Mart — difference from data warehouse, subject-oriented
  • Schema Types — Star schema, Snowflake schema, Fact tables, Dimension tables
  • Metadata — technical, business, operational metadata
🖥️Shell Programming5%
  • Shell Basics — bash scripting, shebang, script execution
  • Variables — env variables, local variables, special variables ($0, $1, $#, $?)
  • Control Flow — if/elif/else, case statements
  • Loops — for, while, until loops; break and continue
  • Functions — defining, calling, return values in bash
  • UNIX Commands — grep, awk, sed, find, chmod, ps, top, pipe operators

Sample Question Formats

Phase II IT paper heavily tests code tracing, debugging, and dry-run outputs. Practice these question types.

Code Dry Run — Java
What is the output of the following Java code?

int[] arr = {5, 3, 8, 1, 9};
for(int i=0; i<arr.length-1; i++){
  for(int j=0; j<arr.length-i-1; j++){
    if(arr[j] > arr[j+1]){
      int t=arr[j]; arr[j]=arr[j+1]; arr[j+1]=t;
    }
  }
}
System.out.println(arr[2]);
A) 3
B) 5 ✓ (bubble sort → [1,3,5,8,9] → index 2 = 5)
C) 8
D) 1
SQL Query Output
Table EMPLOYEE has columns: EmpID, Name, Dept, Salary. Which query returns department-wise average salary only for departments having more than 3 employees?
A) SELECT Dept, AVG(Salary) FROM EMPLOYEE WHERE COUNT(*) > 3 GROUP BY Dept
B) SELECT Dept, AVG(Salary) FROM EMPLOYEE GROUP BY Dept HAVING COUNT(*) > 3 ✓
C) SELECT Dept, AVG(Salary) FROM EMPLOYEE HAVING COUNT(*) > 3
D) SELECT Dept, AVG(Salary) FROM EMPLOYEE GROUP BY Dept WHERE COUNT(*) > 3
Data Structure — Linked List
A singly linked list has nodes: 10 → 20 → 30 → 40 → NULL. After executing: reverse(head), what is the output of printList(head)?
A) 10 → 20 → 30 → 40
B) 40 → 30 → 20 → 10 ✓
C) 40 → 20 → 30 → 10
D) NULL → 10 → 20 → 30 → 40
Networking Concept
Which layer of the OSI model is responsible for end-to-end error detection, flow control, and reliable data transfer between two devices?
A) Network Layer (Layer 3)
B) Transport Layer (Layer 4) ✓ — TCP provides reliable, ordered delivery
C) Data Link Layer (Layer 2)
D) Session Layer (Layer 5)

Exam Strategy & Tips

Specific to Phase II IT paper — where most candidates lose marks

01

Master Code Tracing, Not Just Concepts

Phase II tests your ability to mentally execute code and predict output — not just whether you know the concept. Practice tracing at least 10 programs per day with pen and paper.

02

DS First — 40% of Phase II

Data Structures alone carry 40% of Phase II Paper 2. Prioritize: Linked List, Binary Tree, BST, Heap, and Hashing above everything else in the last month.

03

SQL Query Output Questions

Don't just write queries — practice reading a query and mentally determining its output set. Practice with real table data. GROUP BY + HAVING combinations are tested frequently.

04

Negative Marking = Skip Strategy

With ¼ negative marking, skipping an uncertain question is better than guessing randomly. If you can eliminate 2 options, attempt it. Otherwise, skip. Practice this discipline in mocks.

05

English Descriptive Matters

Phase II Paper 1 is 1/3rd of your score. Write one essay daily. Topics rotate around: digital finance, cybersecurity, fintech regulation, SEBI's role in investor protection. Know these well.

06

Know SEBI's IT Policies for Interview

Read SEBI's Cyber Security & Cyber Resilience Framework for MIIs. Know key IT regulations SEBI has issued — algo trading guidelines, data localization, cloud computing in capital markets.

07

Build a Complexity Cheat Sheet

Tabulate time and space complexity for every sorting algorithm, data structure operation, and graph algorithm. Memorize it. This alone answers 8–10 questions directly.

08

Phase I is Just a Gate — Clear It Efficiently

Don't over-prepare for Phase I. It only requires qualifying marks. Put 70% of energy into Phase II preparation from the beginning. Phase I IT paper covers the same topics as Phase II, just less deep.
